# DevMapper Build Intent Document
## Version 1.0 — Foundation
### Date: 2026-03-22

---

## Core Vision

DevMapper becomes Africa's default infrastructure for tracking, verifying, and monetizing development impact.

- Not a dashboard.
- Not a reporting tool.
- **A system of record + intelligence layer for development activity.**

---

## Core User Personas

### 1. Field Reporter (Grassroots)
- **Who:** NGOs, volunteers, project officers
- **Needs:** Easy reporting, credibility, recognition

### 2. Verifier (Expert Layer)
- **Who:** Academics, auditors, consultants
- **Needs:** Trust system, incentives, workflow tools

### 3. Organizations (Paying Customers)
- **Who:** NGOs, Corporates, Governments, DFIs
- **Needs:** ESG reporting, SDG alignment, impact verification, audit trails

### 4. Ecosystem Players (Future Monetization)
- **Who:** Carbon developers, climate funds, Article 6 traders, donors

---

## Core Value Proposition

| User | Value |
|------|-------|
| Individuals | Recognition + certification |
| NGOs | Structured reporting |
| Corporates | ESG compliance + storytelling |
| Governments | National SDG tracking |
| Carbon players | Verified ground data |

---

## Success Metrics (North Star)

1. Verified Reports
2. Verification Rate (%)
3. Org Subscriptions
4. Cost per Verified Data Point
5. Time to Verification
6. Retention (Reporters + Orgs)

---

## Product Guardrails

### What We Are NOT Building
- No full project management tool (Not Asana)
- No generic social network
- No custom ERP systems for NGOs
- No blockchain-first product (can be added later)
- No heavy AI chat UI (focus on structured AI)

### What We ARE Building
- Structured reporting + verification system
- Lightweight AI-assisted workflows
- Trust layer for development data
- Scalable API-first infrastructure
- Future-ready for: Carbon markets, ESG compliance, Article 6
